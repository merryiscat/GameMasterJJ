프로젝트 디렉토리를 정리한다.

## 점검 항목

### 1. 불필요한 파일 탐지
- `__pycache__/` 디렉토리 잔존 여부
- `.playwright-mcp/` 로그/스냅샷 파일
- 루트에 산재한 스크린샷(*.png) 등 임시 파일
- 사용되지 않는 스크립트 (다른 코드에서 import/호출 안 되는 .py)
- 폐기된 venv 디렉토리
- 빈 디렉토리

### 2. 레거시 코드 탐지
- `app/` 내 어디서도 import 되지 않는 모듈 확인
- `scripts/` 내 현재 파이프라인에서 사용하지 않는 스크립트 확인
- 대체된 파일 (v1 → v2 등) 잔존 확인

### 3. .gitignore 점검
- git 추적 중이지만 .gitignore에 포함되어야 할 파일 확인
- 추적 불필요한 로그, 캐시, 임시 파일

### 4. git 추적 정리
- 위에서 발견한 불필요 파일이 git에 추적 중이면 `git rm --cached`로 제거 제안
- 삭제 전 반드시 목록을 보여주고 사용자 확인 받기

### 5. 00_old 이관
- 불필요 파일은 삭제하지 않고 `00_old/`로 이동
- 카테고리별 하위 폴더로 분류 (screenshots, legacy_code 등)
- `00_old/INDEX.md`에 이관 내역 기록 (원래 위치, 대체된 것, 비고)
- 버전 업데이트 시 폐기된 접근 방식을 INDEX.md 버전 히스토리에 기록
